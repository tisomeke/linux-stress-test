# 🚀 Universal Linux Stress-Tester (Bash)

Универсальный скрипт на Bash для проведения стресс-тестирования процессора (CPU) при помощи архиватора **p7zip** и видеокарты (GPU) при помощи бенчмарка **FurMark**. Скрипт поддерживает автоматическое определение оборудования, динамический поиск датчиков (включая AMD GPU) и расчет средних показателей.

## 🛠 Возможности
* **Три режима работы:** Раздельное тестирование CPU или GPU, а также комбинированный тест (`all`) для проверки блока питания.
* **Настройка времени:** Задание любой длительности теста через флаг `-t` (в секундах).
* **Универсальность:** Автоматический поиск температурных датчиков в `/sys/class/hwmon/` (совместимо с Intel, AMD).
* **Умное логирование:** Запись системной информации, температур, частоты GPU и системной нагрузки (Load Average) в ровную таблицу каждые 10 секунд.
* **Аналитика (AVR):** Автоматический расчет средних значений температуры, частоты и нагрузки по завершении теста.

## 📦 Установка зависимостей

Для работы скрипта требуются `p7zip`, `furmark`, утилиты мониторинга датчиков и консольный калькулятор `bc` (обязателен для расчета AVR).

### Arch Linux / CachyOS
```bash
sudo yay -S p7zip furmark bc lm_sensors # либо через paru вместо yay
sudo sensors-detect  # Выполните один раз для инициализации датчиков
```

### Debian / Ubuntu / Mint
```bash
sudo apt update
sudo apt install p7zip-full bc lm-sensors
# FurMark устанавливается отдельно (рекомендуется bin-версия или через Flathub)
```

### Fedora / Red Hat / CentOS
```bash
sudo dnf install p7zip p7zip-plugins bc lm_sensors
```

## 🚀 Использование

**0. Быстрый старт**
```bash
git clone https://github.com/tisomeke/linux-stress-test
cd linux-stress-test
```

**1. Дайте права на исполнение:**
```bash
chmod +x stress.sh
```

**2. Запуск тестов:**
По умолчанию (если не указывать время) тест длится **300 секунд** (5 минут). Для изменения времени используйте флаг `-t`.

| Команда | Описание |
| :--- | :--- |
| `./stress.sh cpu` | Тест только процессора на 5 минут (100% нагрузка на все потоки). |
| `./stress.sh gpu` | Тест только видеокарты на 5 минут. |
| `./stress.sh all` | **Полный стресс-тест на 5 минут.** Максимальная нагрузка на БП и охлаждение. |
| `./stress.sh -t 60 all` | Полный стресс-тест длительностью **ровно 60 секунд**. |

## 📊 Интерпретация логов
Логи сохраняются в папку `logs/` в формате `stress_РЕЖИМ_ДАТА.log`. 

Файл содержит стартовую информацию о вашем железе и таблицу со следующими колонками:
* **Time:** Время с начала теста.
* **CPU Temp:** Температура процессора (норма до 80-85°C).
* **GPU Temp:** Температура видеокарты (норма до 85°C).
* **GPU Clock:** Текущая частота видеочипа (позволяет отследить сброс частот из-за перегрева).
* **Load:** Нагрузка на систему (Load Average). При тесте CPU должна стремиться к количеству логических потоков вашего процессора.

**В конце таблицы выводится строка `AVR`** — это средние значения (Average) за всё время проведения теста. Они помогают понять реальную картину работы системы, сглаживая кратковременные скачки температур или просадки частот.
